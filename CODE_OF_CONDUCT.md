# Code of Conduct

## Purpose

This project is maintained with an enterprise-style collaboration standard focused on professionalism, clarity, and respectful technical discussion.

Participation in this repository, its issues, pull requests, reviews, and related discussions requires compliance with this Code of Conduct.

## Expected Behavior

Participants are expected to:

- communicate respectfully and professionally
- focus on technical substance, reproducibility, and user impact
- give actionable, specific review feedback
- assume good faith while maintaining engineering rigor
- keep discussions relevant to the repository and the change under review
- respect licensing, attribution, confidentiality, and security boundaries

## Unacceptable Behavior

The following behavior is not acceptable:

- harassment, intimidation, threats, or personal attacks
- discriminatory, hateful, or degrading language
- deliberate disruption of discussions, reviews, or collaboration workflows
- sharing confidential, private, or security-sensitive information without authorization
- knowingly misrepresenting test results, provenance, licensing, authorship, or technical findings
- spam, off-topic promotion, or repeated low-value noise in issues or pull requests

## Review and Collaboration Standards

To keep contributions effective and reviewable:

- keep pull requests focused and scoped
- document validation steps accurately
- disclose breaking changes, security considerations, and license-sensitive updates clearly
- avoid broad rewrites when an incremental change is sufficient
- update documentation when user-facing behavior changes

## Maintainer Responsibilities

Maintainers may:

- request scope reduction or clarification
- ask contributors to improve documentation or validation details
- reject contributions that do not meet repository standards
- remove content that violates this Code of Conduct or creates legal, security, or operational risk

## Enforcement

Project maintainers are responsible for interpreting and enforcing this Code of Conduct.

They may take any action considered appropriate, including:

- requesting edits
- locking or closing discussions
- rejecting pull requests
- restricting participation

## Reporting

If you experience or observe conduct that violates this policy, report it privately to the maintainers or repository owner.

When reporting, include:

- a concise description of the incident
- relevant links or screenshots
- timestamps if available
- any immediate safety, legal, or security concerns

## Scope

This Code of Conduct applies to:

- repository discussions
- issues
- pull requests
- code review comments
- documentation discussions
- other project-managed communication spaces

## Good Engineering Culture

A strong contribution culture for this repository means:

- clear English communication
- reproducible technical claims
- respectful disagreement
- careful handling of licensing and third-party sources
- consistent attention to user safety, operational reliability, and maintainability
